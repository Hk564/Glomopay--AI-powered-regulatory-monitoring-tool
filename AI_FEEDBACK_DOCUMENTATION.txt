1. RELEVANCE SCORE FEEDBACK
----------------------------
Users can approve or disagree with AI's relevance score

Quick Actions (1-2 clicks):
  ✓ "Yes, correct" - Approve current score (1 click)
  ✗ "No, incorrect" - Select correct score (2 clicks)

If disagreed:
  - User selects correct score: HIGH / MEDIUM / LOW / NOT_RELEVANT
  - Optional: Provide reason (future enhancement)

2. ANALYSIS QUALITY FEEDBACK
----------------------------
Users can rate if analysis was useful

Quick Actions:
  👍 "Helpful" - Mark as useful (1 click)
  👎 "Needs Improvement" - Provide details (2-3 clicks)

If not useful, select issue types:
  □ Summary unclear
  □ Impact assessment incorrect
  □ Action items missing or incomplete
  □ Too generic / not specific enough
  □ Missed a key point
  
Optional: Add comment for additional details

=================================================================================
📊 DATABASE SCHEMA
=================================================================================

Table: ai_feedback

Fields:
  - id (UUID, primary key)
  - ai_analysis_id (FK to ai_analysis, unique)
  - regulatory_update_id (FK to regulatory_updates)
  - feedback_type (relevance / analysis_quality / both)
  
  Relevance Feedback:
  - relevance_approved (boolean)
  - correct_relevance_score (HIGH/MEDIUM/LOW/NOT_RELEVANT)
  - relevance_disagreement_reason (text)
  
  Analysis Quality Feedback:
  - analysis_useful (boolean)
  - analysis_issue_types (JSONB array)
  - analysis_comment (text)
  
  Metadata:
  - user_id (for future multi-user support)
  - created_at (timestamp)

Constraints:
  - UNIQUE(ai_analysis_id) - One feedback per analysis
  - Updates allowed (users can change their feedback)

=================================================================================
🔌 API ENDPOINTS
=================================================================================

POST /api/ai-feedback/relevance
--------------------------------
Submit relevance score feedback

Request:
{
  "ai_analysis_id": "uuid",
  "approved": true/false,
  "correct_score": "HIGH" (if not approved),
  "reason": "optional text" (if not approved)
}

Response:
{
  "success": true,
  "message": "Relevance feedback submitted successfully"
}


POST /api/ai-feedback/analysis-quality
---------------------------------------
Submit analysis quality feedback

Request:
{
  "ai_analysis_id": "uuid",
  "useful": true/false,
  "issue_types": ["summary_unclear", "too_generic"],
  "comment": "optional comment"
}

Response:
{
  "success": true,
  "message": "Analysis quality feedback submitted successfully"
}


GET /api/ai-feedback/stats
---------------------------
Get aggregated feedback statistics

Response:
{
  "total_feedback": 10,
  "relevance_feedback": {
    "total": 10,
    "approved": 7,
    "disapproved": 3,
    "approval_rate": 70.0,
    "suggested_correct_scores": {
      "MEDIUM": 2,
      "LOW": 1
    }
  },
  "analysis_quality_feedback": {
    "total": 10,
    "useful": 8,
    "not_useful": 2,
    "useful_rate": 80.0,
    "common_issues": {
      "too_generic": 1,
      "missed_key_point": 1
    }
  }
}


GET /api/ai-feedback/{ai_analysis_id}
--------------------------------------
Get existing feedback for specific analysis

Response:
{
  "success": true,
  "has_feedback": true/false,
  "data": { feedback object } or null
}

🎨 UI/UX IMPLEMENTATION
=================================================================================

Location: AI Analysis Detail Page (after clicking on regulatory update)

Design Principles:
  ✓ Quick feedback (1-2 clicks max)
  ✓ No long forms
  ✓ Integrated into existing UI
  ✓ Clear visual feedback
  ✓ No page reload required

UI Components:

1. Relevance Score Feedback Box
   - Shows current AI score with color coding
   - Two buttons: "✓ Yes, correct" | "✗ No, incorrect"
   - If incorrect: Shows 4 score options (HIGH, MEDIUM, LOW, NOT_RELEVANT)
   - Clicking score submits feedback instantly

2. Analysis Quality Feedback Box
   - Two buttons: "👍 Helpful" | "👎 Needs Improvement"
   - If not helpful: Shows checkboxes for issue types
   - Optional comment textarea
   - Submit button to send feedback

3. Feedback Submitted State
   - Green success message
   - "✓ Thank you for your feedback!"
   - Hides feedback forms after submission

Visual Features:
  - Color-coded relevance scores
  - Smooth transitions
  - Loading states
  - Success/error messages
  - Hover effects
